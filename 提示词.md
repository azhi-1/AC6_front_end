# Task
 **HTML 状态栏界面**需要悬浮在游戏页面上，实时读取并显示最新的 Assistant (AI) 楼层的游戏状态变量 (`stat_data`)。

# Technical Environment & Global Functions
**注意：以下函数是宿主环境直接提供的全局函数，无需 import，直接调用即可。**

## 1. 基础消息获取
`getLastMessageId()`
- **功能**: 获取当前聊天记录中最后一条消息的 ID（数字）。

`getChatMessages(range, options)`
- **功能**: 获取聊天消息对象数组。
- **参数**:
    - `range`: 字符串或数字。例如 `'0-{{lastMessageId}}'` (获取全部) 或 `-1` (获取最新一条)。
    - `options`: 对象 `{ role: 'assistant' | 'user' }`，用于筛选角色。
- **返回**: 消息对象数组，每个对象包含 `message_id`, `role`, `message` (内容) 等字段。

## 2. 变量系统 (MVU Framework)
`getVariables(option)`
- **功能**: 获取原生变量表。
- **参数**: `option` 对象。常用 `{ type: 'message', message_id: number }` 获取指定楼层的局部变量。
- **返回**: 包含所有变量的对象。

`Mvu.getMvuData(option)`
- **功能**: 如果环境支持 MVU 框架，优先使用此函数获取处理后的变量数据。
- **参数**: 同 `getVariables`，如 `{ type: 'message', message_id: 'latest' }`。
- **返回**: `{ stat_data: { ... } }` 结构的数据。

# Data Fetching Logic (Critical)
为了获取最新的游戏状态，请严格遵守以下逻辑编写 JavaScript 代码：
1.  **获取锚点**: 调用 `getLastMessageId()` 获取最大 ID。
2.  **回溯查找**: 从最新楼层开始，向前查找最后一条 `role` 为 `'assistant'` (AI) 的消息。
    *   *原因*: 只有 AI 发出的消息才包含最新的世界状态更新，用户的消息不包含状态。
3.  **读取变量**:
    *   优先检查 `window.Mvu` 是否存在。如果存在，调用 `Mvu.getMvuData({type: 'message', message_id: targetMsgId})`。
    *   如果不存在，降级调用 `getVariables({type: 'message', message_id: targetMsgId})`。
4.  **提取数据**: 从返回结果中提取 `stat_data` 对象。如果提取失败，使用兜底的 Mock 数据。
